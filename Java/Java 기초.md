# 자바란?

- 자바란? 객체지향 프로그래밍 언어
- 프로그래밍 : 컴퓨터 프로그램을 만다는 일
- 프로그램 : 컴퓨터에서 특정 작업을 수행하는 명령어들의 모음
- 프로그래밍 언어
    - 명령어를 만들기 위해 프로그래밍 언어를 사용
    - 사람이 컴퓨터와 의사소통 할 수 있는 도구
    - 특정 작업의 흐름대로 컴퓨터가 수행할 수 있도록 프로그래밍 언어를 작성
- 소스 코드 : 프로그래밍 언어로 작성한 명령어 - 사람이 작성
- 소스 파일 : 소스 코드로 이루어진 파일
- 기계어 : 0과 1로 이루어진 이진코드 , 컴퓨터가 실제로 이해하는 언어
- 컴파일러 : 프로그래밍 언어로 작성된 명령어를 기계어로 번역
- 자바가 어떻게 실행되는가?
    - 자바 컴파일러 (javac)
        - 자바 언어로 작성된 소스파일(.java)를 바이트 파일(.class)로 컴파일
    - 바이트 파일(.class)
        - 자바에서 소스 코드를 바이트 코드로 컴파일한 파일
        - 바이트 파일 = 바이트 코드의 집합
        - JVM이 읽기(번역)에 수월하도록 만들어 놓는 형태의 파일
            
           
    - JVM (Java Virtual Machine)
        - JVM이 바이트 파일을 기계어로 번역
        - 프로그램을 실행
    - 자바의 특징
        - 운영체제에 상관없이 모든 환경에서 실행 가능
        - 안정성
            - 안정적으로 프로그램을 구동
            - 가비지 컬렉터가 메모리를 자동으로 효율적인 관리
        - 풍부한 오픈 소스
        - 객체 지향 프로그래밍 언어
            - 설계 측면에서 “객체 지향 프로그래밍”
            - 자바는 대표적인 객체 지향 프로그래밍 언어
            - 개발 유지보수, 확장성이 좋음

## 자바 코드 스타일과 주석문

### 자바 코드 스타일

- 세미콜론 ;
    - 코드 문장의 마침표이자 구분자 역할
    - 컴파일러가 세미콜론까지 하나의 문장으로 해석함
    - 문장이 끝났음을 표시해야 하고, 안하면 컴파일 에러가 발생함
- 들여쓰기 [tap]
    - 왼쪽에서 일정 간격을 띄워 문장을 작성
    - 영역 구분을 위해 사용
    - 동일한 영역은 동일한 들여쓰기를 가짐
    - 코드를 읽을 때 구분이 잘 되는 장점
    - 간격은 일반적으로 space 2칸 또는 4칸
- 중괄호 { }
    - 중괄호 한 상으로 반드시 사용
    - 여는 중괄호는 문장 오른쪽 끝에 위치
    - 중괄호 한 쌍은 같은 영역이기 때문에 들여쓰기 간격이 동일함

### 주석문

- 라인 주석 //

- 블록 주석 /* */

- JavaDoc 주석 /** */

# 자바의 변수, 타입, 연산자

- 변수란? 특정 데이터 값을 저장하는 메모리 공간에 붙인 이름

- 변수 선언하기
    - 변수로 저장할 데이터가 어떤 종류 - 타입(자료형) : 데이터 값의 종류를 분류해 놓은 것
    - 변수의 이름 - 변수명
        
    - 변수 여러개 동시에 선언
        - 타입이 동일한 경우 사용할 수 있음
        - 콤마(,)를 사용
            
    - 변수명 규칙
        
        
    - 변수 사용하기
        - 변수 초기화 - 값 대입하기
            - 변수 초기화란? 변수에 값을 대입
            - 대입 연산자(=)를 사용
                
                - 변수 초기화 시점 다르게 가능
                    - 변수를 먼저 선언하고 나중에 초기화 하는 경우
                        - 값 타입을 잘 확인하여 선언
                        - 나중에 다른 타입 값을 넣으면 에러 발생
                        - 동일한 영역에서 같은 변수명으로 재선언X
            - 기본타입
                - 기본타입
                    - 정수
                        - byte, short, int, long : 메모리 사용 크기, 값 범위가 다름
                            - 값 범위에 맞춰 타입 선언
                            - 범위에 맞지 않는 타임에 값을 저장하면 에러 발생!
                            - long에 큰 값을 대입할 때, 정수 값 뒤 l 또는 L을 붙여야 함
                        - 변수에 정수 값을 저장할 때 사용
                        
                    
                    - 문자
                        - char
                        - 변수에 문자 값을 저장할 때 사용
                        - 작은 따옴표(’ ‘)사용
                        - 컴파일러가 유니코드 매핑 변환 후, 유니코드 정수 값을 저장
                            - 유니코드란?
                                - 국제 표준 규약인 문자 표현에 대한 약속
                                - 전 세계의 모든 문자를 컴퓨터에서 일관되게 표현하고 다룰 수 있도록 설계되어 있음
                          
                    - 실수
                        - float, double
                            - float
                                - 소수점 아래 7자리까지
                                - float에 값을 대입할 때 실수 값뒤 f또는 F를 붙여야 함
                        - 변수에 실수 값을 저장할 때 사용
                        
                    - 논리
                        - boolean
                        - 변수에 논리 값을 저장할때 사용

                    - 상수(constant) : 절대 변하지 않는 수

                            - 왼쪽 앞에 final을 추가
                            - 값을 딱 한번만 대입할 수 있음
                            - 값이 한번 정해지면 이후 다른 값으로 변경이 불가능 함
                                
                        - 상수명 명명규칙
                            - 모두 대문자
                            - 단어 두개 이상인 경우, 단어 사이에 언더바를 넣음
                            
                            
                - 기본 타입 변환
                    - 어떤 기본타입 변수를 다른 타입으로 변환
                   
                    
                    - 자동타입변환(묵시적 타입 변환)
                        - 강제로 시키지 않아도 자동으로 타입이 변환되는 것
                        - 특정 조건에서 자동으로 변환됨
                        - 값범위가 작은 타입을 큰 타입으로 대입
                            
                            
                        - 덜 정밀한 타입에서 더 정밀한 타입으로 대입
                            
                            
                        - 연산하는 경우
                            
                        
                    - 강제타입변환(명시적 타입 변환)
                        - 강제로 타입을 변환하는 것
                            
                        - 캐스팅 연산자 괄호 ()를 사용
                            - 소괄호 안에 변환할 타입을 넣어준다.
                            
                            
                            - 주의 : 강제 타입 변환에서는 더 작고, 덜 정밀한 타입으로 바꾸면서 완전히 다른 값으로 바뀔 수 있음 → 위 사진의 밑에 코드
                        - 괄호여 안에는 변환할 타입을 입력
                    
                    # 연산자와 항
                    
                    - 연산자란?
                        - 연산에 사용하는 기호
                        - 기호 단독으로 사용할 수 없음
                        - 연산을 목적으로 특정 값과 함께 기호를 사용함
                    - 항 (operand) - 위에 나오는 특정 값
                        - 연산에서 사용되는 값
                    
                        
                        - 다양한 연산자
                            - 항(operand) 수에 따라 단항, 이항, 삼항 연산자로 나눔
                         
                            
                            - 단항 연산자
                                - 항이 1개일 때 사용하는 연산자
                                - 부호 연산자, 증감 연산자, 논리 부정 연산자
                                    - 부호 연산자
                                        - +(양수),  -(음수)
                                        - 연산자를 항 앞에 위치시킴
                                        - 값을 양수 또는 음수로 만듦
                                        - 연산은 항(연산에 쓰인 값)에 영향을 주지 않음
                                        
                                        
                                    - 증감 연산자
                                        - ++(증가 연산자) : 값을 1 증가시킴
                                        - — (감소 연산자) : 값을 1 감소시킴
                                        - 연산 결과가 항(연산에 쓰인 값)에 영향을 줌
                                        - 항 앞에 위치 or 항 뒤에 위치 (연산 순서가 달라짐)
                                        
                                     
                                    - 논리 부정 연사자
                                        - !
                                        - boolean타입 변수에만 사용
                                        - 항 앞에 위치
                                        - 연산은 항에 영향을 주지 않기 때문에 변수에 대입해서 사용
                                        
                            - 이항 연산자
                                - 항이 2개일 때 사용하는 연산자
                                - 산술 연산자, 대입 연산자, 비교 연산자, 논리 연산자
                                    - 산술 연산자
                                        - + , - , * , / , %(나머지)
                                    - (복합) 대입 연산자
                                        - +=,  -=, *=, /=, %=
                                        - 산술 연산한 결과를 왼쪽 변수에 대입
                                            
                                           
                                    - 비교 연산자
                                        - < , ≤ , > , ≥ , == , ! =
                                        - 두 개 항의 값을 비교하는 연산자
                                        - 비교 연산 후 결과값으로 true 또는 false를 반환
                                      
                                    - 논리 연산자
                                        - 논리 명제와 유사
                                        - && (논리곱, AND) : 2개 항이 둘 다 참일때만 true
                                        - || (논리합, OR) : 1개라도 참이면 true
                                            
                                            
                            - 삼항 연산자
                                - 항을 3개 사용하는 연산자
                                - 조건 연산자
                                    - 조건을 확인하는 연산자
                                    
                        
                                    
                        - 연산의 방향과 우선순위
                            - 우선순위 : 누가 먼저 연산처리되는지
                            - 연산방향 : 우선 순위가 같을 때 어느 쪽 방향으로 연산 처리 해야 하는지
                                
                               
                                
                                - 우선순위
                                    1. 산술 (*)
                                    2. 산술 (+,-)
                                    3. 비교
                                    4. 조건
                                    
                               

# 제어문

## if문

- if문
    - “만약 어떤 조건일 때”를확인해서 실행 제어하는 구문
    - if에 괄호로 조건식을작성
    - 조건식의 겨과에 따라 블록 실행을 제어
    - 조건식이 true이면블록 실행, false이면 블록 실행 안함
        - 블록이란? 같은 중괄호 한 쌍 안에 있는 영역
        
        
        1. score변수가 80보다 큰 값인지 비교
        2. 조건식 false 반환
        3. if문의 다음 문장 수행
    - 조건이 여러개 일때? else와 else if문 활용
        - else문
            - if문 블록에 이어서 작성
            - 조건식이 false일때 else문 블록 실행
            - if문 없이 else문 단독 사용 불가
        - else if문
            - 조건이 여러 개인 경우 사용
            - if문 블록에 이어서 else if문 작성
            - else if문은여러 개 추가 가능
            - if문 없이else문 단독 사용 불가
            
            
            
- switch문
    
    : 동일한 값을여러 조건으로 확인할 때 유용
    
     - 변수에 일치하는 값이 없으면 default문 실행
